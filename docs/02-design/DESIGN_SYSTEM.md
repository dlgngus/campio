# Campio Design System v1.0

## Overview

Campio uses a Framer-inspired dark canvas design system.

The product should feel:
- premium
- minimal
- focused
- student-friendly
- opportunity-first

The dominant surface is a near-black canvas.
Most UI is white typography, charcoal cards, soft borders, pill CTAs, and sparse accent blue.

## Design Principles

### 1. Opportunity First

The opportunity card is the hero.
Do not let decorative UI overpower opportunity information.

### 2. Dark Canvas Only

No light mode.
All pages use the dark canvas.

### 3. One Primary Action

Every screen has one dominant action.

Examples:
- Landing: Get Started
- Detail: Apply Now
- Explore: Filter/Search
- Onboarding: Continue

### 4. Premium but Useful

Framer-like visual style, but with slightly higher information density for browsing many opportunities.

### 5. Cards, Not Tables

Most student-facing opportunity information should be card-based.
Admin pages may use tables.

## Visual Language

- Large display type
- Tight letter spacing
- Rounded cards
- White pill buttons
- Charcoal secondary buttons
- Gradient spotlight cards used sparingly
- Accent blue only as signal color

## Do

- Use large whitespace
- Use clear hierarchy
- Use reusable components
- Use skeleton loading
- Use responsive cards
- Keep body copy short

## Do Not

- Do not create light pages
- Do not use blue CTA buttons
- Do not use many accent colors
- Do not use heavy shadows
- Do not overuse gradients
- Do not create dense old-school admin UI for student-facing pages
